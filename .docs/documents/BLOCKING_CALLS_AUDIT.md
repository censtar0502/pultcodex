# Аудит блокирующих вызовов

## Назначение

Этот документ фиксирует текущие блокирующие места в коде, чтобы их постепенно убрать.

## Политика

С текущего этапа проекта:
- новые блокирующие вызовы добавлять запрещено
- при острой необходимости нужно отдельно согласовать это с пользователем

## Найденные блокирующие места

### `ssd1322.c`

- `HAL_SPI_Transmit(..., HAL_MAX_DELAY)` для команд и данных
- `HAL_Delay(20)` в аппаратном reset дисплея

### `eeprom_at24.c`

- `HAL_I2C_Mem_Read(...)`
- blocking `HAL_I2C_Mem_Write(...)` сохранён только как legacy helper
- рабочая запись конфигурации переведена на `HAL_I2C_Mem_Write_IT(...)`
  с фоновым page-by-page service

## Комментарий

Часть этих вызовов появилась на раннем этапе bring-up и диагностики железа.

Они не должны становиться основой окончательной архитектуры.

## План замены

### OLED

- переход на queued/DMA-передачу
- неблокирующий display task
- при необходимости неблокирующий reset/init state machine

### EEPROM

- отдельный state machine для чтения/записи
- запись страниц по шагам
- фоновая обработка через периодическую задачу
- runtime-запись уже переведена на этот путь
- остаётся убрать blocking startup-read при инициализации

### Логи

- только через ring buffer и сервисную задачу

## Статус

Документ является рабочим контрольным списком.
После удаления очередного блокирующего места его нужно обновлять.

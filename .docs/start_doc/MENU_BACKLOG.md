# Menu Backlog

Дата фиксации: 2026-03-25

## Что нужно реализовать по документации

Меню по проекту разделено на два уровня:

- пользовательское меню
- административное меню с PIN

## Пользовательское меню

Согласованный минимальный состав по `MENU_SPEC.md`:

1. `Цена`
- цена `TRK1`
- цена `TRK2`

2. `Сумматоры`
- сумматор `TRK1`
- сумматор `TRK2`

3. `Последний чек`
- перепечатать `TRK1`
- перепечатать `TRK2`

4. `Статус`
- состояние `TRK1`
- состояние `TRK2`

5. `Вход в администрирование`

## Административное меню

### Настройки `TRK1`

- протокол
- адрес
- тип топлива
- цена
- slow polling
- fast polling

### Настройки `TRK2`

- протокол
- адрес
- тип топлива
- цена
- slow polling
- fast polling

### Принтер

- печать вкл/выкл
- количество копий
- заголовок
- подвал
- тест печати

### Система

- яркость дисплея
- звук / buzzer
- дата и время
- PIN администратора

### Сервис

- тест клавиатуры
- тест дисплея
- тест EEPROM
- тест связи с `TRK1`
- тест связи с `TRK2`
- просмотр ошибок

### Сброс

- сброс пользовательских настроек
- заводские настройки

## Что уже есть в коде

Пока реализован только эксплуатационный минимум:

- `TRK1 ON/OFF`
- `TRK2 ON/OFF`
- `TRK1 PRICE`
- `TRK2 PRICE`

Это временный стартовый setup-экран, а не полное финальное меню.

## Что логично делать дальше

После текущего архитектурного рефакторинга логично идти так:

1. привести текущий `SETUP MENU` к пользовательскому меню
2. добавить экран `Статус`
3. добавить вход в PIN
4. после этого наращивать административные разделы

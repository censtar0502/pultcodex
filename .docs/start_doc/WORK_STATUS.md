## Текущее фактическое состояние кода

### Transport / polling

- модуль `Core/Src/trk_probe.c` пока временно совмещает:
  - polling статуса
  - разбор ответа `S`
  - runtime-модель канала
  - минимальную UI-логику выбора канала и редактирования цены

Это временный шаг.

Антидребезг связи:
- единичный timeout или битый кадр больше не переводит канал сразу в `Not Connect!`
- канал считается offline только после `5` подряд ошибок связи
- первый же валидный ответ сразу возвращает канал в `online`

Дальше этот код нужно разнести ближе к целевой архитектуре:
- transport
- parser
- FSM
- UI

Что уже сделано для уменьшения смешения обязанностей:
- `transport/parser` физически вынесены из тела `trk_probe.c` в
  `Core/Src/trk_probe_transport.inc`
- `ui-controller` и обработка клавиатуры вынесены в
  `Core/Src/trk_probe_ui.inc`
- `trk_probe.c` теперь в основном держит runtime/model, конфиг и общую связку

Это пока ещё один translation unit, но код уже разделён по зонам ответственности
без риска для сборки `STM32CubeIDE`.

Текущий фактический протокольный цикл для `GasKitLink`:
- дежурный режим: `S`
- после старта и во время отпуска: `S -> L -> R`
- idle polling остаётся `300 ms`
- active transaction polling ускорен до `120 ms`
- при `S81`: один запрос `T`
- при `S90`: если final data ещё не были получены, сначала `T`, потом `N`

### FSM

Введены базовые состояния:
- `OFFLINE`
- `IDLE`
- `CALLING`
- `AUTH_WAIT`
- `STARTED`
- `PAUSED`
- `FUELLING`
- `FINISHING`
- `FINISHED_HOLD`
- `ERROR`

Пока эти состояния обновляются только из ответов статуса `S`.

Дополнительно runtime уже хранит:
- `transaction_pending`
- `live_money`
- `live_volume_cl`
- `final_money`
- `final_volume_cl`
- `final_data_ready`
- `transaction_id`

### UI

На экране сейчас показываются:
- в основном режиме:
  - `Off`, если канал выключен
  - `Not Connect!`, если канал включён, но связи нет
  - `A:...`, `L:...` или `H`, если канал включён и связь есть
- рядом с заголовком канала теперь есть служебная индикация:
  - мигающий `UP` при `S21`, если транзакция ещё не отправлена
  - мигающая `*` при `S31`
  - постоянный `UP` во время `STARTED/PAUSED/FUELLING`
  - `END` во время `FINISHING/FINISHED_HOLD`
- фокус активной ТРК показывается отдельным маркером
- выбор активной ТРК:
  - `G` -> `TRK1`
  - `H` -> `TRK2`
- после загрузки конфигурации активная ТРК теперь нормализуется:
  - если сохранённый/дефолтный фокус указывает на выключенный канал, фокус переносится на первый включённый
- клавиша `B` на главном экране циклически меняет режим отпуска активной ТРК
- цифры `0..9` на главном экране теперь редактируют preset активной ТРК
- `.` на главном экране используется для объёма в режиме `L`
- `E` на главном экране удаляет последнюю цифру preset активной ТРК
- под основным значением мелким шрифтом показывается вычисляемая вторая величина
- основной текст рабочего состояния увеличен относительно обычного 8x8
- во время отпуска вместо preset показываются live-значения
- после получения `T` показываются final-значения
- при первом новом вводе preset после завершённой транзакции held final data сбрасываются,
  чтобы новое значение не склеивалось со старым
- в режиме полного бака во время отпуска и после финала больше не показывается пустое `H`:
  оператор видит реальный объём и сумму
- основной крупный online-текст теперь рисуется стабильным размером без переключения
  между 8x8 и 16x16
- после возврата в `S10` final data теперь удерживаются на экране, пока оператор
  не начнёт новый ввод или не сменит режим

### Меню

Добавлено минимальное меню настроек.

Пока в нём есть:
- `TRK1 ON/OFF`
- `TRK2 ON/OFF`
- `TRK1 PRICE`
- `TRK2 PRICE`

Клавиши:
- `F` -> вход в меню
- `A/B` -> перемещение по пунктам
- `K` -> подтвердить / изменить
- `E` -> выход / отмена / очистка

### Цена

Цена уже заведена как часть runtime-данных канала.

Текущая логика:
- дефолтная цена обоих каналов `1100`
- текущая модель приведена ближе к `GasKitLink`
- цена хранится в двух видах:
  - строка для UI и EEPROM, чтобы не терять формат `00.45` / `99.99` / `9999`
  - raw значение `pppp` для будущих команд `V/M`
- редактирование теперь открывается из меню
- во время ввода цены:
  - отображается старая цена (`OLD`)
  - текущее редактируемое значение отображается отдельно (`NEW`)
  - цифры `0..9` вводят значение
  - `.` вводит десятичную точку
  - `K` сохраняет
  - `E` удаляет символ, а на пустом буфере отменяет редактирование
  - после сохранения выводится подтверждение `Price saved`
  - при ошибке формата или диапазона выводится `Bad price`
  - допустимы форматы с суммарно не более 4 цифр без точки, например:
    - `00.45`
    - `99.99`
    - `9999`
- цена и состояние `ON/OFF` временно сохраняются в `AT24C256`
- при старте они загружаются из EEPROM, если блок валиден по magic/version/CRC
- исправлен баг сохранения цены:
  - раньше при `K` строка ввода очищалась раньше, чем копировалась в persistent price text
  - из-за этого `OLD` мог стать пустым, а EEPROM сохраняла пустую цену

### Preset

- в режиме `A` вводится сумма до `999999`
- в режиме `L` вводится объём до `900.00 л`
- в режиме `H` preset рассчитывается автоматически как полный бак
- для полного бака учитываются оба ограничения протокола:
  - максимум `900 л`
  - максимум `999999` по сумме

### Запуск транзакции

- `K` на главном экране теперь отправляет команду старта для активной ТРК
- для `GasKitLink` реализован первый боевой шаг:
  - `M1;mmmmmm;pppp` в режиме суммы
  - `V1;llllll;pppp` в режиме объёма и полного бака
- отправка разрешена при online-связи и статусах:
  - `IDLE`
  - `CALLING` (`S21`)
- полный `TRK TX/RX` лог снова включён в USB CDC

### Live / final exchange

- во время реальной транзакции уже есть live polling:
  - `S`
  - `L`
  - `R`
- ответы `L...` обновляют текущий объём
- ответы `R...` обновляют текущую сумму
- при `S81` отправляется `T` один раз
- ответ `T...` сохраняет финальные объём/сумму/ID транзакции
- при `S90` без уже полученного `T` сначала запрашиваются final data
- затем отправляется `N` один раз
- после возврата в `S10` runtime live/final данные очищаются

## Что ещё НЕ сделано

- отдельный transport layer как постоянный модуль
- полноценный parser ответов `GasKitLink`
- отдельный `channel_fsm.c`
- полноценное меню
- расширенное меню настроек
- пауза / продолжение `B/G` во время уже идущей транзакции
- печать / итоговый чек
- удержание финальных данных отдельной пользовательской логикой
- явная реализация inter-byte timer через `TIM3 1 ms`

## Особо важное замечание

`TIM3` уже настроен как `1 ms` база, но в текущем коде ещё не используется как
явный механизм межбайтового таймаута `<= 3 ms`.

Временная EEPROM-персистентность сейчас использует blocking `AT24` driver как
bring-up компромисс. По документации позже это надо вынести в отдельный
неблокирующий EEPROM service.

Это не забыть при следующем transport-этапе.

## Мелкие, но важные изменения текущего шага

- логика выбора активной ТРК перенесена с `A/B` на `G/H`
- роль клавиши `F` изменена: теперь это вход в меню
- роль клавиши `E` изменена: теперь это выход / отмена / очистка
- стартовый основной экран приведён ближе к согласованной логике оператора
- режим отпуска на главном экране теперь меняется по `B`
- увеличен шрифт основного рабочего значения
- добавлен live/final обмен и экранная индикация этапов транзакции
- `S21` теперь не только виден на экране, но и допускает последующую отправку транзакции по `K`
- если пульт стартует уже на `S90`, он теперь старается не закрыть транзакцию вслепую,
  а сначала забрать final data через `T`
- в активной транзакции live-поля обновляются быстрее за счёт ускоренного polling interval
- после успешного старта транзакции буфер ручного ввода preset очищается,
  чтобы следующая доза начиналась с нового ввода
- в ходе ревью найден и исправлен хвост:
  раньше один и тот же runtime reset стирал и активное состояние, и held final data
- дополнительно перед следующим этапом выполнен безопасный внутренний рефакторинг,
  чтобы transport/parser/ui больше не были свалены в одном длинном куске кода

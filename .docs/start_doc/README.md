# Start Doc

Эта папка нужна как короткая оперативная память проекта на случай потери контекста
в чате.

Здесь нужно фиксировать:
- что уже реально работает на железе
- какой этап сейчас активен
- какие последние изменения внесены в код
- что проверять первым после перезапуска
- какой следующий шаг согласован

## Точка входа

Сначала читать:

1. `CURRENT_STAGE.md`
2. `WORK_STATUS.md`

Потом уже переходить к основной документации:

1. `..\MASTER_INDEX.md`
2. `..\PULT_PROJECT_CONTEXT.md`
3. `..\ARCHITECTURE.md`
4. `..\FOR_CODEX_PROTOCOL_ABSTRACTION.md`
5. `..\IMPLEMENTATION_RULES.md`
6. `..\NEXT_ACTIONS.md`

## Правило обновления

После каждого существенного шага по коду обновлять минимум:
- `CURRENT_STAGE.md`
- `WORK_STATUS.md`

Если изменился порядок разработки или сменился активный этап, это обязательно
нужно отразить здесь сразу.

## Обязательное правило с 2026-03-24

Отныне, помимо `commit` и `push`, нужно вести подробную документацию в этой
папке.

Фиксировать нужно не только крупные этапы, но и любые заметные мелочи:
- небольшие изменения логики
- промежуточные аппаратные наблюдения
- результаты тестов
- временные допущения
- найденные риски
- изменения порядка работ

Если после шага в коде документация в `start_doc` не обновлена, шаг считается
незавершённым.

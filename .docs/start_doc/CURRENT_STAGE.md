# Current Stage

Дата фиксации: 2026-03-25

## Где мы сейчас

Проект находится на раннем рабочем этапе с уже живым стартом транзакции:

- базовый двухканальный обмен с реальными ТРК подтверждён
- `TRK1` и `TRK2` работают как независимые физические каналы
- на текущем стенде оба канала используют адрес `0x00 0x01`
- текущий режим обмена:
  - дежурный polling `S` каждые `300 ms`
  - в активной транзакции цикл `S -> L -> R`
  - после `S81` один запрос `T`
  - после `S90` один запрос `N`
- приём сделан через `UART DMA + IDLE`

## Активный этап разработки

Текущий этап:

- минимальная FSM канала
- `IDLE` является состоянием FSM, а не внешним режимом
- поверх этого строится базовый UI и работа с клавиатурой
- preset уже вводится на главном экране
- старт транзакции уже отправляется
- сейчас идёт этап live/final обмена и экранной индикации хода отпуска

## Согласованный порядок работ

1. Устойчивый двухканальный `IDLE polling`
2. Минимальная FSM канала
3. Базовый рабочий экран и фокус активной ТРК
4. Установка и редактирование цены по каждой ТРК
5. Меню и навигация
6. Ввод preset по сумме или объёму
7. Формирование кода транзакции
8. Live/final polling и закрытие транзакции
9. Дальнейшее меню и расширение UI

## Что уже сделано в коде

Последний подтверждённый шаг:

- в коде введено состояние канала (`IDLE`, `CALLING`, `AUTH_WAIT`, и др.)
- статус `S` теперь отображается в channel state
- на OLED есть рабочий стартовый экран двух ТРК по схеме:
  - `Off`
  - `Not Connect!`
  - `A: 0`
- `Not Connect!` теперь показывается только после серии подряд ошибок связи
- текущий порог потери связи: `5` подряд невалидных или потерянных ответов
- цена на главном экране не отображается
- активная ТРК теперь выбирается клавишами:
  - `G` -> `TRK1`
  - `H` -> `TRK2`
- если активен только один канал, выбор второй ТРК с клавиатуры игнорируется
- добавлено минимальное меню настроек на клавише `F`
- отмена / выход / очистка перенесены на клавишу `E`
- навигация по меню:
  - `A/B` -> перемещение
  - `K` -> выбор / подтверждение
  - `E` -> назад / отмена
- на главном экране `B` теперь циклически меняет режим активной ТРК:
  - `A` -> на сумму
  - `L` -> на объём
  - `H` -> полный бак
- в дежурном режиме цифры теперь работают на главном экране для активной ТРК
- в режиме `A` вводится сумма, а под основным значением показывается ожидаемый объём
- в режиме `L` вводится объём, а под основным значением показывается ожидаемая сумма
- в режиме `H` рассчитывается полный бак с ограничениями `900 л` и `999999`
- `K` на главном экране теперь отправляет код транзакции `GasKitLink`:
  - `M1;mmmmmm;pppp` для режима `A`
  - `V1;llllll;pppp` для режимов `L/H`
- старт транзакции теперь допустим и из `S21`, если пистолет уже поднят
- при активной транзакции добавлен live polling:
  - `S`
  - `L`
  - `R`
- при `S81` теперь один раз запрашивается `T`
- при `S90` теперь один раз отправляется `N`
- на главном экране добавлена служебная индикация состояния рядом с `TRK`:
  - мигающий `UP` для `S21` без транзакции
  - мигающая `*` для `S31`
  - постоянный `UP` во время `S41/S61/S51/S71`
  - `END` в финальной фазе `S81/S90`
- на главном экране во время отпуска отображаются live-значения
- после получения `T` отображаются final-значения
- протокольный `TRK TX/RX` лог снова включён для обеих ТРК
- основной текст рабочего состояния увеличен за счёт масштабирования шрифта
- в редактировании цены:
  - старая цена показывается отдельно как `OLD`
  - новое значение вводится отдельно как `NEW`
  - после `K` сначала показывается подтверждение сохранения, затем выполняется выход
  - разрешены только цифры и одна десятичная точка
  - для `GasKitLink` цена теперь хранится как raw поле `pppp` и отдельно как строка отображения
  - строка цены сохраняет операторский формат, например `00.45`, `99.99`, `9999`
  - в код транзакции `GasKitLink` позже должна уходить нормализованная 4-значная цена без точки
 - цена и состояние каналов теперь временно сохраняются в `AT24` и загружаются после перезагрузки
 - в меню строки `TRK1/TRK2` показывают текущее состояние `ON/OFF`, а не статичное `ENABLE`

## Что следующим проверять на железе

1. Оба канала продолжают стабильно отвечать в polling
2. Для включённого, но не отвечающего канала показывается `Not Connect!`
3. Для отключённого в меню канала показывается `Off`
4. Для включённого и отвечающего канала показывается `A: 0`
5. `G/H` корректно выбирают активную ТРК, если включены оба канала
6. `F` открывает меню
7. `E` работает как выход / отмена / очистка
8. Timeout одного канала не ломает второй канал
9. `B` меняет режим отпуска по кругу только у активной ТРК
10. в редактировании цены видна старая цена
11. после сохранения цены видно подтверждение `Price saved`
12. `NEW` при входе пустой и не копирует `OLD`
13. некорректная цена даёт сообщение `Bad price` и не сохраняется
14. после перезагрузки сохраняются цена и `ON/OFF`
15. в меню у `TRK1/TRK2` видно текущее состояние `ON/OFF`
16. одиночный сбой связи не должен сразу давать `Not Connect!`
17. цифры на главном экране относятся только к активной ТРК
18. в режиме `A` и `L` под основным значением видна вычисленная вторая величина
19. после `K` в логах виден старт `M/V` и дальнейший цикл `S/L/R`
20. при `S21` на экране виден мигающий `UP`
21. после авторизации `S31` на экране видна мигающая `*`
22. во время отпуска на экране виден `UP` без мигания
23. при завершении отображается `END`
24. после `S90` отправляется `N`
25. после возврата в `S10` канал снова показывает обычный дежурный экран

## Важные проектные правила

- читать всю документацию перед следующим большим этапом
- не дублировать код под `TRK1` и `TRK2`
- не выносить протокольную логику в UI
- не использовать новые blocking-вызовы
- временный `AT24` допустим как bring-up шаг, но позже заменить на неблокирующий EEPROM service
- работать CubeMX-safe
- помимо `commit/push`, всегда вести подробную документацию в `start_doc`,
  включая даже мелкие изменения и промежуточные наблюдения
